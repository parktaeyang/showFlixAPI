# 요구사항 기반 구현 계획서 생성

## 요구사항 파일 내용
!`cat local-notes/요구사항.txt 2>/dev/null || echo "요구사항.txt 파일이 없습니다."`

## 수행 절차

1. 위 요구사항 내용을 분석합니다.
2. 프로젝트 코드베이스를 깊이 탐색하여 현황을 파악합니다.
   - 레거시 코드가 관련된 경우 `legacy/` 디렉토리를 반드시 참고합니다.
   - 기존 구현 패턴(아키텍처, 레이어 구조, 네이밍 규칙 등)을 파악합니다.
3. 파일명을 `local-notes/YYYYMMDDNN-요구사항요약-plan.md` 형식으로 생성합니다.
   - YYYYMMDD: 오늘 날짜 (예: 20260402)
   - NN: 같은 날짜 내 2자리 시퀀스 번호 (01부터 시작). `local-notes/` 디렉토리에서 같은 날짜(YYYYMMDD)로 시작하는 `*-plan.md` 파일 수를 확인하고 다음 번호를 부여합니다.
   - 요구사항요약: 요구사항 내용을 핵심 키워드 중심으로 짧게 요약 (예: 역할변경저장오류, 엑셀다운로드개선, 로그인기능추가)
   - 예시: `2026040201-역할변경저장오류-plan.md`, `2026040202-엑셀다운로드개선-plan.md`

## plan 파일 작성 형식

```markdown
# [기능명] 구현 계획

---

## 1. 요구사항
- 요구사항.txt 내용을 정리하고 구체화

---

## 2. 현황 분석
- 관련 레거시 코드 분석 (해당하는 경우)
- 현재 프로젝트 아키텍처/패턴 분석
- 관련 테이블/API/화면 구조

---

## 3. 변경 계획
- 새로 생성할 엔드포인트/파일 목록
- 수정할 기존 파일 목록

---

## 4. 상세 구현 방법
### Step 1: ...
### Step 2: ...
### Step 3: ...
(각 단계별 구체적인 코드 예시 포함)

---

## 5. 변경 범위 요약
| 구분 | 파일 수 | 내용 |
|------|--------|------|

---

## 6. 추가 고려사항
(보안, 마이그레이션, 의존성, 성능 등 검토 필요 사항)
```

## 주의사항
- 프로젝트를 충분히 탐색한 후 계획서를 작성하세요.
- 기존 코드 패턴과 일관성을 유지하는 방향으로 계획하세요.
- 추가 고려사항에는 사용자가 결정해야 할 선택지를 명확히 제시하세요.
